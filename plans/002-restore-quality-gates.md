## Why this matters

This is a full-TypeScript monorepo with **no working quality gate of any
kind**: the committed `.eslintrc.json` is orphaned (ESLint is not a dependency
of any package and no `lint` script exists anywhere), the `husky` config in
root `package.json` uses the dead husky≤4 format (husky is not installed, no
`.husky/` directory exists, and the lint-staged glob `src/**/*` matches nothing
from the repo root), there is no `tsc --noEmit` typecheck script in any
package, and two generated files (`platform/app/commit.txt`,
`platform/app/version.txt`) perpetually dirty the working tree because they're
not gitignored. Restoring these gates gives every subsequent change (human or
agent) a fast, local correctness signal — several other plans in this directory
assume it.

## Steps

### Step 1: Gitignore the generated platform/app artifacts

Append to `.gitignore`:

```
# Generated by version.mjs (platform/app copies; root copies are tracked release artifacts)
platform/app/commit.txt
platform/app/version.txt
```

**Verify**: `git check-ignore platform/app/commit.txt platform/app/version.txt` → prints both paths, exit 0. Then `git status --porcelain | grep -c "platform/app/\(commit\|version\).txt"` → `0`.

### Step 2: Working pre-commit hook (husky 9 + fixed lint-staged glob)

1. Add root devDependencies: `husky` (^9), `lint-staged` (^15). Run
   `pnpm install --no-frozen-lockfile`.
2. Delete the dead `"husky"` key (lines 116-120) from root `package.json`.
3. Replace the `"lint-staged"` config with a glob that actually matches this
   monorepo's layout, prettier-only for now (lint-staged ≥10 auto-stages, so no
   `git add`):

```json
"lint-staged": {
  "{platform,extensions,modes}/**/src/**/*.{js,jsx,ts,tsx,json,css}": [
    "prettier --write"
  ]
}
```

4. Add `"prepare": "husky"` to root scripts and create `.husky/pre-commit`
   containing exactly:

```
pnpm exec lint-staged
```

**Verify**: run `pnpm run prepare` → exit 0 and `.husky/pre-commit` exists. Then do a live test: append a space-mangled line to a throwaway copy — `cp extensions/default/src/index.ts /tmp/x.ts` is NOT valid (out of tree); instead: `git stash list` must be clean, then `printf 'const  x = 1\n' >> platform/app/src/sanity.test.js && git add platform/app/src/sanity.test.js && pnpm exec lint-staged` → exit 0 and `git diff --cached platform/app/src/sanity.test.js` shows the line reformatted by prettier. Then restore: `git restore --staged platform/app/src/sanity.test.js && git checkout -- platform/app/src/sanity.test.js`.

### Step 3: Install ESLint and add a `lint` script

The legacy config extends `react-app`, which drags a large peer-dependency
tree from the CRA era. Drop it; keep the rest.

1. Edit `.eslintrc.json`: remove `"react-app"` from `extends`; remove the
   Cypress globals block (`cy`, `Cypress`, etc.) only if plan 012 has already
   landed — otherwise leave it.
2. Add root devDependencies (ESLint 8.x line, matching the legacy config
   format): `eslint@^8.57.0`, `@typescript-eslint/parser@^7`,
   `@typescript-eslint/eslint-plugin@^7`, `eslint-plugin-react@^7`,
   `eslint-plugin-import@^2`, `eslint-plugin-tsdoc@^0.3`,
   `eslint-plugin-prettier@^5`, `eslint-config-prettier@^9`. Run
   `pnpm install --no-frozen-lockfile`.
3. Add root script:
   `"lint": "eslint --ext .js,.jsx,.ts,.tsx platform/app/src platform/core/src platform/ui-next/src platform/i18n/src extensions modes --cache"`
   (deliberately excludes legacy `platform/ui` and `platform/cli`).
4. Run `pnpm run lint 2>&1 | tail -5` and record the error/warning totals.
   - If errors ≤ 50: fix nothing; record the count in `plans/README.md` notes.
   - If errors > 50: identify the top offending rules
     (`pnpm run lint -- -f json | ...` or read the tail summary) and set AT
     MOST the 5 noisiest rules to `"warn"` in `.eslintrc.json` `rules`, with a
     `// TODO(baseline)` comment, until errors ≤ 50. Record which rules were
     downgraded.

**Verify**: `pnpm exec eslint --version` → prints `v8.x`. `pnpm run lint` → exits with ≤ 50 errors (warnings unlimited); record the numbers.

### Step 4: Typecheck script that doesn't fight the root tsconfig

1. Create `tsconfig.typecheck.json` at repo root:

```json
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "noEmit": true,
    "emitDeclarationOnly": false,
    "declaration": false,
    "checkJs": false,
    "skipLibCheck": true
  }
}
```

   (`checkJs: false` because the root config's `checkJs: true` over untyped
   legacy JS would produce thousands of diagnostics; `skipLibCheck` to avoid
   churning on third-party d.ts.)
2. Add root devDependency `typescript` (match the version other packages use —
   check `platform/core/package.json` first; if none is pinned anywhere, use
   `~5.5`). `pnpm install --no-frozen-lockfile`.
3. Add root script: `"typecheck": "tsc -p tsconfig.typecheck.json"`.
4. Run it and count errors: `pnpm run typecheck 2>&1 | grep -c "error TS"`.
   - 0 errors: done.
   - 1–300 errors: do NOT fix them; record the count and the 10 most frequent
     error codes in `plans/README.md` notes as the baseline.
   - More than 300 errors: STOP condition.

**Verify**: `pnpm run typecheck; echo "exit=$?"` runs to completion (any exit
code) and the error count is recorded. `pnpm -r run test:unit:ci` still exits 0.
