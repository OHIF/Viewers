## Direction findings (not planned — maintainer options)

Recorded for context; these are strategy calls, not queued work:

- **Finish the three in-flight migrations to their finish lines.** ui→ui-next
  (plan 014), yarn→pnpm docs/tooling (plans 002, 010), Cypress→Playwright
  (plan 012) are each ~stalled at the last mile; the plans above cover the
  mechanical parts, the memos cover the judgment parts.
- **Consolidate onto one CI system with an enforced gate.** GitHub Actions as
  canonical, with typecheck + lint + unit + Playwright as one green signal;
  retire the CircleCI/Cypress/paid-context stack. Certain redundancy; the
  target provider is a maintainer decision. A dedicated plan was intentionally
  not written (needs the maintainer's provider choice first); plans 002, 012,
  015 lay the groundwork.
- **Mature the StudyPrefetcherService.** A small TODO cluster in a
  performance-critical, half-finished prefetch surface
  (`platform/core/src/services/StudyPrefetcherService/`). Thin evidence — an
  investigate/spike, not a build. Not planned.

## Findings considered and rejected

- **`defineExtension()` helper to deduplicate the 14 extensions' module
  boilerplate** — conflicts with the repo's explicit "never modify core
  architecture" rule for a marginal gain; the boilerplate is stable and
  low-churn. Not worth the core-surface risk.
- **Rollbar client token in `platform/app/public/html-templates/rollbar.html`**
  — publicly-embeddable `post_client_item` token by design; only residual risk
  is event-spam, handled by Rollbar-side quotas. Not a finding.
- **`textarea.innerHTML` in `DataRow.tsx`** — standard HTML-entity-decode idiom
  (reads `.value`, never re-inserts as HTML). Safe.
- **`pnpm audit --prod`** — returned no critical/high advisories affecting the
  production PWA runtime at `973631b7e`. No dependency-vulnerability plan
  needed now.
- **Pub-sub subscription-leak sweep** — audited and found clean: services
  (`SegmentationService.destroy()`) and hooks (`useMeasurements`,
  `ActiveViewportBehavior`, `useViewportRendering`) unsubscribe correctly. No
  plan needed.
